    树莓派3B 和 树莓派3B+ 安装 Ubuntu mate16.04版本的那些事 ？

    嵌入式 > 操作系统 树莓派3B 树莓派3B+ Ubuntu mate 16.04
    Created At : 2024-04-23 03:37
    Views 👀 :
    1. 这个问题太奇怪了

    这个问题太奇怪了

    如果你是用的树莓派3B+的板子，想要安装Ubuntu mate16.04的系统，直接下载本文提供的百度网盘链接，适合树莓派3B+板子的系统。系统是armhf架构。

    硬件环境和软件环境背景介绍
    （1）硬件环境 是 树莓派3B+，确认过硬件是没有任何问题的（不要问原因，因为同一张SD卡，装其它版本的系统就没有问题，如Ubuntu mate 18.04）
    （2）Windows 10系统上做烧录工作，系统镜像文件是Ubuntu mate 16.04，烧录软件有用:
    a. 树莓派官方的软件：raspberry pi imager
    b. refus 软件
    c. balenaEtcher 软件

    问题现象
    a. 问题背景
    实验室有两台堆满了灰的turtlebot小车，刚接手时，从来没有弄过ROS系统和树莓派方面的东西，从接手小车时，确定车辆型号 到 根据手册熟悉车辆上的硬件，起初只是在一辆车作参考来学习，以及配环境，没有关注到两个车上的树莓派板子型号不同（一起买的车，没想到型号会不同）。在根据教程配置环境过程中教程链接，碰巧拿来学习的小车上的树莓派板子是 “树莓派3B”，在配置树莓派环境时，安装 Ubuntu mate 16.04系统没有任何问题，没有遇到彩虹屏问题（该问题的另一个表现就是板子上的ACT指示灯不亮，代表SD不活动，即系统没有启动）。

    b.问题产生
    当搭载 “树莓派3B” 的turtlebot小车上的环境配置好后，就在另一台车辆上配置，以为轻松能够完成，没想到竟然在给 上边的树莓派配置环境时，碰到了彩虹屏问题，就觉得很奇怪，网上给出的原因如下
    网络上的原因归类
    （1）系统镜像可能已经损坏，重新下载镜像；
    （2）烧录软件问题，换烧录软件；
    （3）树莓派板子坏了；
    （4）系统烧录后，SD中系统的启动文件有问题（上述3个原因无效，请注意该原因）
    试了很多方法都不行，也通过其它一些方法间接证明了不是（1-3）问题造成的，并且得出树莓派3B+的板子可以装 Ubuntu mate16.04以后的系统，就是不能装官方上的 Ubuntu mate16.04，通过修改启动文件（原因4），可以进入系统（后文有如何修改），但是最后发现不能使用Wifi（后文有介绍）。要想解决该问题，就需要正确的修改启动文件，下边的百度网盘链接是一个适合 树莓派3B+的 Ubuntu mate16.04系统镜像（感谢好心人提供）。

    官网下载的镜像始终有错误，该镜像不知道具体是从何而来，我知道的原地址为原文地址，文中提到修改SD卡启动文件，会出现Wifi不能用的情况（亲测过，确实会这样，开启wifi时，直接提示找不到wlan0设备，即使使用 ifconfig -a,也查不到任何网卡信息），具体如何改的，我是拿的 Ubuntu mate 18.04 的启动文件中的一些 替换掉 Ubuntu mate 16.04，原因就是前者可以在 “树莓派3B+”上正常运行，不过比较卡，应该是挺卡的，不推荐安装该版本，也试过其它一些系统的启动文件，正常开机注册用户后，都显示的是Wifi不能用，真的是大坑（勿踩）。
